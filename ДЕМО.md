# Демонстрация работоспособности платформы MT (YTrans)

Документ для защиты: как подготовить базу, под какими аккаунтами входить,
по шагам — что нажимать, и готовый текст рассказа.

---

## 1. Подготовка базы данных

База уже содержит полный демонстрационный датасет (37 заказов, 11 пользователей,
переписка, претензии, документы, расчёты ставок). Чтобы привести её к чистому
«свежему» состоянию прямо перед показом, выполните сброс одной командой.

**Локально:**
```
cd backend
npm install        # один раз
npm run seed:demo  # сброс + загрузка демо-данных
```

**На Railway (production):**
```
cd backend
railway run npm run seed:demo
```

Скрипт идемпотентный: очищает таблицы и загружает данные заново, а все даты
считаются относительно текущего момента, поэтому заказы и сообщения всегда
выглядят актуальными. Статус «Спор» и блок «Ставки» уже учтены.

> Сайт: **https://dip-production-7155.up.railway.app**
> При первом запуске сервера база инициализируется и наполняется автоматически.

## 3. Сценарий показа (по шагам)

### Блок A. Брокер — создание заказа с расчётом ставки (ключевая новая функция)

1. Войдите как **`broker@mt.by`** / `demo1234`.
2. В левом меню нажмите **«Заказы»**.
3. Нажмите синюю кнопку **«+ Новый заказ»** (справа сверху).
4. В блоке **«Отправка»** введите город — например, **Минск**.
5. В блоке **«Доставка»** введите город — например, **Варшава**.
   - 👉 Обратите внимание: в блоке **«Ставки»** поле **«Расстояние (км)»**
     заполнилось **автоматически** по введённым городам.
6. Прокрутите к блоку **«Ставки» → «Калькулятор ставки»**:
   - Выберите **Тип ТС** (например, «Тентованный»),
   - Введите **Вес** (например, 20),
   - Выберите **Срочность** (например, «Стандарт»).
7. Нажмите **«Рассчитать ставку»** — появится сумма, ставка за км и рыночный
   диапазон за 90 дней.
8. Нажмите **«+ В ставки»** — расчёт добавится в список ставок заказа.
   При желании добавьте свою ставку кнопкой **«+ Добавить ставку»**
   (Перевозчик, Сумма).
9. Нажмите **«Создать заказ»**.
   - 👉 Сразу откроется окно **«Автоматическое назначение водителя»** —
     система подобрала подходящих водителей по расстоянию, рейтингу и доступности.
10. Выберите водителя и нажмите **«Предложить»** (или **«Авто-назначить лучшего»**).

### Блок B. Калькулятор ставок и история (раздел «Ставки»)

1. В меню брокера нажмите **«Ставки»**.
2. Вкладка **«Калькулятор»**: введите маршрут (Минск → Берлин), параметры →
   **«Рассчитать»**. Покажите состав цены и рыночные данные.
3. Нажмите **«Сохранить в историю»**.
4. Вкладка **«История ставок»** — список сохранённых расчётов, выгрузка в Excel.

### Блок C. Диспетчер — трекинг и управление

1. Выйдите и войдите как **`dispatcher@mt.by`** / `demo1234`.
2. **«Трекинг»** — карта с движущимися водителями (обновление в реальном времени).
3. **«Заказы»** — вкладки статусов: «Новый», «Назначен», «Забран», «Доставлен»,
   «Оплачен», **«Спор»**, «Архив». Откройте вкладку **«Спор»** — заказы со спорными
   ситуациями (бывш. «Клейм»).
4. Откройте любой заказ → покажите смену статуса, контакты, транспорт.

### Блок D. Водитель — рейсы и подтверждения

1. Войдите как **`driver@mt.by`** / `demo1234`.
2. **«Заказы»** — назначенные рейсы.
3. Откройте рейс → **«Подтвердить доставку»**, при необходимости — **«Инцидент»**.

### Блок E. Сквозные функции (на любом аккаунте)

- **«Сообщения»** — деловая переписка брокер ↔ диспетчер ↔ водитель.
- **«Контакты»** — справочник, рейтинги (звёзды), отзывы.
- **«Бухгалтерия»** — суммы, выплаты.
- **«Документы»** — CMR, ТТН, сроки.
- **«Претензии»** — спорные заказы.
- **`admin@mt.by` → «Администрирование»** — права ролей, аудит действий.

## 4. Текст рассказа (для защиты)

> **Вступление.**
> «Тема работы — веб-платформа для брокерских грузоперевозок. Это система
> с разграничением ролей: администратор, диспетчер, брокер и водитель.
> Фронтенд — React, бэкенд — Node.js + Express, база данных — MySQL,
> отслеживание в реальном времени — через WebSocket. Приложение развёрнуто
> в облаке и доступно по ссылке.»

> **Вход и роли.**
> «Вход выполняется по логину и паролю, токен — JWT. В зависимости от роли
> пользователь видит свой набор разделов. Сейчас я зайду как брокер.»

> **Создание заказа (главная функция).**
> «Брокер формирует заказ. Я указываю город отправления и город назначения —
> и обратите внимание: расстояние между городами подставляется автоматически,
> по координатам городов с поправкой на дорожный коэффициент. Это избавляет
> от ручного ввода и ошибок.»
>
> «Далее — встроенный калькулятор ставки. Он учитывает тип транспорта, вес,
> срочность и сезонность. Я нажимаю «Рассчитать» — система выдаёт сумму,
> стоимость за километр и рыночный диапазон по похожим маршрутам за последние
> 90 дней. Рассчитанную ставку я одной кнопкой добавляю в заказ; можно добавить
> и собственные ставки от перевозчиков.»
>
> «После создания заказа автоматически запускается подбор водителя: система
> предлагает подходящих исполнителей по расстоянию, рейтингу и доступности —
> брокеру остаётся выбрать и отправить предложение.»

> **Диспетчер и трекинг.**
> «Диспетчер видит карту с водителями в реальном времени и управляет статусами
> заказов на всех этапах — от «Нового» до «Оплачен», включая статус «Спор» для
> спорных ситуаций.»

> **Водитель.**
> «Водитель видит свои рейсы, подтверждает доставку и может сообщить об инциденте.»

> **Сквозные возможности.**
> «В системе также есть встроенный мессенджер, справочник контактов с рейтингами
> и отзывами, бухгалтерия, документооборот (CMR, ТТН), претензии, аналитика,
> а у администратора — управление правами ролей и журнал аудита всех действий.»

> **Заключение.**
> «Таким образом, платформа закрывает полный цикл работы брокера: расчёт ставки,
> создание заказа, подбор и назначение водителя, отслеживание перевозки,
> документооборот и взаиморасчёты — в едином веб-интерфейсе.»
